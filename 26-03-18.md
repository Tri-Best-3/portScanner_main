# 변경사항 정리 (2026-03-18)

## 목적

- 시나리오를 공용 구조로 전환
- 기존 코드 오염 없이 시나리오 주입/실행 분리

## 주요 변경

1. backend 시나리오 레지스트리 자동화

- 파일: `backend/app/services/scenario_service.py`
- 기존 하드코딩 `_SCENARIOS` 제거
- `scenarios/*/metadata.json` 자동 스캔 방식으로 변경
- 시나리오 메타 필드(`name`, `target_hint`, `container_name`, `script_path`, `description`) 기반 등록/조회
- 실행 시 metadata 기준으로 대상 컨테이너에 스크립트 주입 후 재시작

2. 공용 주입 스크립트 추가

- 신규: `scenarios/scenario_setup.sh`
- 신규: `scenario_setup.sh`(루트 wrapper)
- 사용법: `./scenario_setup.sh <scenario_name> [container_name]`
- `container_name` 미입력 시 `metadata.json.container_name` 자동 사용

3. `redis_drift` 시나리오 표준화

- 수정: `scenarios/redis_drift/scenario.sh`
- 동작: 60~180초 랜덤 간격으로 6379(redis), 22/80(nc) 포트 상태 랜덤 변경
- 안정성: PID 파일 관리 + PID 1 보호 로직 적용 (메인 프로세스 kill 방지)
- 신규: `scenarios/redis_drift/metadata.json`

4. 시나리오 작성 가이드 문서화

- 신규: `scenarios/README.md`
- 공용 폴더 구조, 작성 규칙, metadata 예시, 실행 방법 정리

5. 기존 개별 setup 정리

- 삭제: `scenarios/redis_drift/setup.sh`

## 변경 파일 목록

- 수정: `backend/app/services/scenario_service.py`
- 수정: `scenarios/redis_drift/scenario.sh`
- 신규: `scenarios/redis_drift/metadata.json`
- 신규: `scenarios/scenario_setup.sh`
- 신규: `scenario_setup.sh`
- 신규: `scenarios/README.md`
- 삭제: `scenarios/redis_drift/setup.sh`

## 실행 및 확인 예시

```sh
./scenario_setup.sh redis_drift
docker exec -it vuln-redis-4-unacc sh -lc "ps -ef | grep -E 'scenario|redis|nc' && netstat -tpln"
```
