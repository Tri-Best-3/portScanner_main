# 변경사항 정리 (2026-03-20)

## 목적

- 대시보드에서 시나리오를 수동 입력이 아닌 드롭다운으로 선택
- 검증 탭을 수동 기록형이 아닌 자동 조회형으로 전환

## 주요 변경

1. 시나리오 선택 방식 변경 (드롭다운)

- 파일: `dashboard/app.py`
- `GET /api/v1/scenarios` 호출 함수 추가
  - `get_scenarios(url)`
- 스캔 실행 UI에 `Scenario` 드롭다운 추가
  - `(none)` + API에서 내려온 시나리오 목록 표시
- 선택한 시나리오를 workflow 실행 payload에 포함
  - `POST /api/v1/workflows/run` 호출 시 `scenario` 필드 전달

2. 검증 결과 탭 자동화

- 파일: `dashboard/app.py`
- `GET /api/v1/verifications/{scan_id}` 호출 함수 추가
  - `get_verifications(url, scan_id)`
- 결과 화면 탭 구성 변경
  - 기존: AI/포트/취약점/로그/JSON
  - 변경: AI/포트/취약점/**검증 결과**/로그/JSON
- 검증 결과 표시 방식
  - 목록 테이블: verification_id, template_id, method, status, target, created_at
  - 상세 확장: evidence, raw_output

3. 대시보드 렌더 함수 시그니처 조정

- 파일: `dashboard/app.py`
- `render_dashboard(data)` -> `render_dashboard(data, backend_url)`
- 검증 API 조회에 `backend_url` 사용

## 변경 파일

- 수정: `dashboard/app.py`

## 테스트/확인

1. 정적 확인
- `dashboard/app.py` 파이썬 파싱/컴파일 확인 완료

2. 런타임 확인
- Docker로 `dashboard` 이미지 빌드
- 컨테이너 기동 후 Streamlit 정상 부팅 로그 확인
- `http://localhost:18501` 접속 시 `HTTP 200` 확인
- 테스트 컨테이너 정리 완료

## 비고

- 시나리오 목록/검증 결과는 backend API 응답에 의존함
- backend 미기동 또는 API 오류 시, 대시보드에 경고/안내 문구 표시
